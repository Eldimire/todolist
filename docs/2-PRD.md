## 1. 제품 개요

### 목적
인증된 사용자가 개인별 할일을 카테고리 및 일정(시작일/종료일) 기반으로 등록·관리할 수 있는 웹 애플리케이션

### 배경
- 사용자별 독립적인 할일 관리 서비스의 부재
- 시작일·종료일 기반의 일정 추적 기능이 미흡한 기존 도구의 한계

### 목표
- 1인 개발자가 2일 내에 핵심 기능을 완성 및 배포
- 동시 접속 1,000명을 안정적으로 수용하는 서비스 제공

---

## 2. 타겟 사용자 & 페르소나

### 타겟 사용자
- 학생 (과제·시험·프로젝트 일정 관리)
- 20대~50대 직장인 (업무 태스크 및 개인 일정 관리)

### 핵심 사용 시나리오
- 할일을 카테고리로 분류하여 체계적으로 관리하고 싶은 사용자
- 시작일과 종료일을 지정하여 일정 기반으로 진행 상태를 추적하고 싶은 사용자

## 5. 비기능 요구사항

### 5.1 성능
- 동시 접속 1,000명 처리
- API 응답 시간 95th percentile 500ms 이하
- DB 커넥션 풀 적절히 구성 (pg 라이브러리 풀링 활용)

### 5.2 보안
- 비밀번호 bcrypt 해시 저장
- JWT 기반 인증 (만료 시간 설정 필수)
- 모든 API 엔드포인트에서 본인 데이터만 접근 가능 (BR-02 적용)
- SQL Injection 방지: pg 파라미터 바인딩 사용

### 5.3 호환성
- 웹 우선 (최신 브라우저 Chrome, Edge, Firefox 지원)
- 반응형 웹 UI 적용 (모바일 별도 앱 없음)

### 5.4 국제화 (i18n) `v2 신규`
- 지원 언어: 한국어(ko), 영어(en), 일본어(ja)
- 모든 UI 텍스트는 i18n 리소스 파일로 관리 (하드코딩 금지)
- 언어 전환 시 페이지 새로고침 없이 즉시 반영

## 9. 프로젝트 범위 & 제약사항

### 범위 내 (In Scope)

**v1**
- 사용자 인증 (회원가입·로그인·로그아웃·내 정보 수정)
- 카테고리 CRUD
- 할일 CRUD + 완료 처리
- 할일 목록 필터링 (전체·카테고리별·상태 4종)
- 반응형 웹 UI

**v2**
- 다국어 지원 (한국어·영어·일본어)
- 다크 모드 / 라이트 모드 테마 설정

### 범위 외 (Out of Scope)
- 모바일 네이티브 앱
- 알림·푸시 기능
- 소셜 로그인 (OAuth)
- 할일 공유·협업 기능
- 접근성(Accessibility) 지원

### 제약사항
- ORM(Prisma) 사용 금지, pg 라이브러리 직접 사용
- 1인 개발 체제
- 개발 기간 2일 (v1 기준)
