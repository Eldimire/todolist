## 1. 프로젝트 개요

**TodoList**는 인증된 사용자가 개인별 할일을 카테고리 및 일정(시작일/종료일) 기반으로 등록·관리할 수 있는 웹 애플리케이션이다.

---

## 2. 문제 정의

| # | 문제 |
|---|------|
| 1 | 인증된 사용자 개인별로 할일을 독립적으로 관리할 수 있는 서비스가 부재하다. |
| 2 | 기존 할일 관리 도구에서 시작일과 종료일을 명시적으로 지정하고 일정 기반 상태를 추적하는 기능이 미흡하다. |

---

## 3. 도메인 모델

### 3.1 핵심 엔티티

#### User (사용자)

| 속성 | 타입 | 설명 | 제약 조건 |
|------|------|------|-----------|
| id | UUID | 사용자 고유 식별자 | NOT NULL, PK |
| email | String | 로그인 이메일 (유일값) | NOT NULL, UNIQUE, RFC 5321 형식 |
| password | String | 암호화된 비밀번호 | NOT NULL, 최소 8자 이상 (입력 원문 기준) |
| name | String | 사용자 이름 | NOT NULL, 최대 100자 |
| createdAt | DateTime | 가입 일시 | NOT NULL |
| updatedAt | DateTime | 정보 수정 일시 | NOT NULL |

#### Category (카테고리)

| 속성 | 타입 | 설명 | 제약 조건 |
|------|------|------|-----------|
| id | UUID | 카테고리 고유 식별자 | NOT NULL, PK |
| userId | UUID | 소유 사용자 (FK) | NOT NULL, FK → User.id |
| name | String | 카테고리명 | NOT NULL, 최대 100자 |
| isDefault | Boolean | 기본 카테고리 여부 | NOT NULL, DEFAULT false |
| createdAt | DateTime | 생성 일시 | NOT NULL |

#### Todo (할일)

| 속성 | 타입 | 설명 | 제약 조건 |
|------|------|------|-----------|
| id | UUID | 할일 고유 식별자 | NOT NULL, PK |
| userId | UUID | 소유 사용자 (FK) | NOT NULL, FK → User.id |
| categoryId | UUID | 카테고리 (FK, 미지정 시 기본 카테고리) | NOT NULL, FK → Category.id |
| title | String | 할일 제목 | NOT NULL, 최대 200자 |
| description | String | 상세 내용 (선택) | 최대 2000자, NULL 허용 |
| startDate | Date | 시작일자 | NOT NULL |
| endDate | Date | 종료일자 | NOT NULL, endDate >= startDate |
| isCompleted | Boolean | 완료 여부 | NOT NULL, DEFAULT false |
| createdAt | DateTime | 등록 일시 | NOT NULL |
| updatedAt | DateTime | 수정 일시 | NOT NULL |

### 3.2 엔티티 관계

```
User 1 ──── N Category
User 1 ──── N Todo
Category 1 ──── N Todo
```

## 4. 핵심 비즈니스 규칙

| # | 규칙 |
|---|------|
| BR-01 | 회원가입 후 로그인(인증)을 완료한 사용자만 앱의 모든 기능을 사용할 수 있다. |
| BR-02 | 사용자는 자신의 할일과 카테고리만 조회·등록·수정·삭제할 수 있다. |
| BR-03 | 할일 등록 시 카테고리를 지정하지 않으면 자동으로 '기본' 카테고리가 적용된다. |
| BR-04 | '기본' 카테고리는 각 사용자에게 자동 생성되며 삭제할 수 없다. |
| BR-05 | 종료일자는 시작일자보다 이전일 수 없다. |
| BR-06 | 할일의 진행 상태는 시작일자, 종료일자, 완료 여부를 기반으로 아래와 같이 판단한다. |

### 할일 상태 판단 기준

> "오늘"은 **KST(UTC+9) 기준 당일 날짜(00:00:00 ~ 23:59:59 KST)**를 의미한다.

| 상태 | 조건 |
|------|------|
| 시작 전 | `isCompleted = false` AND `오늘(KST) < startDate` |
| 진행 중 | `isCompleted = false` AND `startDate <= 오늘(KST) <= endDate` |
| 완료 | `isCompleted = true` |
| 기한 초과 | `isCompleted = false` AND `오늘(KST) > endDate` |

## 6. 오류 응답 규약

### 6.1 HTTP 상태 코드 정의

| 상태 코드 | 의미 | 사용 상황 |
|-----------|------|-----------|
| 200 | OK | 조회, 수정, 삭제 등 요청 처리 성공 |
| 201 | Created | 신규 리소스 생성 성공 (회원가입, 할일 등록 등) |
| 400 | Bad Request | 입력값 형식 오류, 필수 필드 누락, 유효성 검사 실패 |
| 401 | Unauthorized | 인증되지 않은 요청 (토큰 없음, 만료, 유효하지 않음) |
| 403 | Forbidden | 인증은 되었으나 해당 리소스에 대한 권한 없음 |
| 404 | Not Found | 요청한 리소스가 존재하지 않음 |
| 409 | Conflict | 리소스 충돌 (이미 사용 중인 이메일 등) |
| 422 | Unprocessable Entity | 비즈니스 규칙 위반 (종료일 < 시작일, 기본 카테고리 수정/삭제 시도 등) |

### 6.2 오류 응답 JSON 스키마

모든 오류 응답은 아래 스키마를 따른다.

```json
{
  "code": "ERROR_CODE",
  "message": "오류에 대한 설명 메시지"
}
```

예시:

```json
{
  "code": "INVALID_DATE_RANGE",
  "message": "종료일자는 시작일자보다 이전일 수 없습니다."
}
```

```json
{
  "code": "UNAUTHORIZED",
  "message": "인증이 필요합니다."
}
```

```json
{
  "code": "FORBIDDEN",
  "message": "해당 리소스에 대한 권한이 없습니다."
}
```

## 7. 인수 기준 (Acceptance Criteria)

> Given-When-Then 형식으로 작성한다. "오늘"은 KST(UTC+9) 기준 당일 날짜를 의미한다.

### F-AUTH-001: 회원가입

**AC-AUTH-001-01: 정상 가입**
- Given: 미가입 사용자가 회원가입 화면에 접근한다.
- When: RFC 5321 형식의 이메일, 8자 이상의 비밀번호, 이름을 입력하고 가입을 요청한다.
- Then: 계정이 생성되고(HTTP 201), 해당 사용자의 기본 카테고리가 자동 생성된다.

**AC-AUTH-001-02: 이메일 형식 오류**
- Given: 사용자가 회원가입 화면에 접근한다.
- When: RFC 5321 형식에 맞지 않는 이메일을 입력하고 가입을 요청한다.
- Then: 가입이 거부되고(HTTP 400), 이메일 형식 오류 메시지가 반환된다.

**AC-AUTH-001-03: 비밀번호 최소 길이 미달**
- Given: 사용자가 회원가입 화면에 접근한다.
- When: 7자 이하의 비밀번호를 입력하고 가입을 요청한다.
- Then: 가입이 거부되고(HTTP 400), "비밀번호는 최소 8자 이상이어야 합니다." 오류 메시지가 반환된다.

---

### F-AUTH-002: 로그인

**AC-AUTH-002-01: 정상 로그인**
- Given: 가입된 사용자가 로그인 화면에 접근한다.
- When: 등록된 이메일과 일치하는 비밀번호를 입력하고 로그인을 요청한다.
- Then: 인증이 성공하고(HTTP 200), 세션(토큰)이 발급된다.

**AC-AUTH-002-02: 비밀번호 불일치**
- Given: 가입된 사용자가 로그인 화면에 접근한다.
- When: 등록된 이메일과 일치하지 않는 비밀번호를 입력하고 로그인을 요청한다.
- Then: 로그인이 거부되고(HTTP 401), 인증 실패 오류 메시지가 반환된다.

**AC-AUTH-002-03: 미가입 이메일 로그인 시도**
- Given: 사용자가 로그인 화면에 접근한다.
- When: 가입되지 않은 이메일로 로그인을 요청한다.
- Then: 로그인이 거부되고(HTTP 401), 인증 실패 오류 메시지가 반환된다.

---

### F-AUTH-003: 로그아웃

**AC-AUTH-003-01: 정상 로그아웃**
- Given: 인증된 사용자가 유효한 세션(토큰)을 보유하고 있다.
- When: 로그아웃을 요청한다.
- Then: 세션(토큰)이 만료 처리되고(HTTP 200), 이후 해당 토큰으로의 요청은 거부된다.

**AC-AUTH-003-02: 미인증 상태에서 로그아웃 시도**
- Given: 사용자가 유효한 세션(토큰)을 보유하지 않은 상태이다.
- When: 로그아웃을 요청한다.
- Then: 요청이 거부되고(HTTP 401), 인증 필요 오류 메시지가 반환된다.

---

### F-AUTH-004: 내 정보 수정

**AC-AUTH-004-01: 정상 수정**
- Given: 인증된 사용자가 내 정보 수정 화면에 접근한다.
- When: 유효한 이름 또는 비밀번호를 입력하고 수정을 요청한다.
- Then: 사용자 정보가 정상 수정되고(HTTP 200), `updatedAt`이 갱신된다.

**AC-AUTH-004-02: 중복 이메일로 변경 시도**
- Given: 인증된 사용자가 내 정보 수정 화면에 접근한다.
- When: 이미 다른 사용자가 사용 중인 이메일로 변경을 요청한다.
- Then: 수정이 거부되고(HTTP 409), "이미 사용 중인 이메일입니다." 오류 메시지가 반환된다.

---

### F-CAT-002: 카테고리 수정

**AC-CAT-002-01: 정상 수정**
- Given: 인증된 사용자가 본인 소유의 일반 카테고리(`isDefault = false`)를 선택한다.
- When: 유효한 카테고리명(100자 이하)으로 수정을 요청한다.
- Then: 카테고리명이 정상 수정된다(HTTP 200).

**AC-CAT-002-02: 기본 카테고리 수정 시도**
- Given: 인증된 사용자가 `isDefault = true`인 기본 카테고리에 대해 수정을 요청한다.
- When: 요청이 처리된다.
- Then: 수정이 거부되고(HTTP 422), "기본 카테고리는 수정할 수 없습니다." 오류 메시지가 반환된다.

---

### F-CAT-003: 카테고리 삭제

**AC-CAT-003-01: 일반 카테고리 삭제 시 하위 Todo 처리**
- Given: 인증된 사용자가 하위 할일이 존재하는 일반 카테고리를 삭제 요청한다.
- When: 삭제가 처리된다.
- Then: 해당 카테고리가 삭제되고(HTTP 200), 하위 할일의 categoryId는 해당 사용자의 기본 카테고리로 자동 재지정된다.

**AC-CAT-003-02: 기본 카테고리 삭제 시도**
- Given: 인증된 사용자가 `isDefault = true`인 기본 카테고리에 대해 삭제를 요청한다.
- When: 요청이 처리된다.
- Then: 삭제가 거부되고(HTTP 422), "기본 카테고리는 삭제할 수 없습니다." 오류 메시지가 반환된다.

---

### F-TODO-001: 할일 등록

## 8. 용어 사전

| 용어 | 설명 |
|------|------|
| 사용자 (User) | 회원가입을 완료하고 로그인한 서비스 이용자 |
| 인증 (Authentication) | 이메일/비밀번호를 통해 사용자 신원을 확인하는 절차 |
| 할일 (Todo) | 사용자가 등록하는 작업 단위. 시작일자·종료일자·완료 여부를 포함한다. |
| 카테고리 (Category) | 할일을 분류하기 위한 그룹 단위 |
| 기본 카테고리 | 카테고리 미지정 시 자동 적용되는 카테고리. 사용자별 자동 생성된다. |
| 시작일자 (startDate) | 할일을 시작하는 날짜 |
| 종료일자 (endDate) | 할일을 완료해야 하는 목표 날짜 |
| 기한 초과 (Overdue) | 종료일자가 지났으나 완료되지 않은 상태 |
| 진행 중 (In Progress) | 시작일자 이후이며 종료일자 이전이고 완료되지 않은 상태 |
| 시작 전 (Not Started) | 시작일자가 오늘 이후인 미완료 상태 |
| 완료 (Completed) | 사용자가 완료 처리한 할일 상태 |
| 오늘 (Today) | KST(UTC+9) 기준 당일 날짜(00:00:00 ~ 23:59:59 KST) |
